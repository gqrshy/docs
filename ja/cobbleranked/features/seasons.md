# シーズン管理

---
**CobbleRanked** > **Features** > **シーズン**
---

自動ローテーションと報酬を伴う時間ベースの競技期間。

---

## 概要

シーズンはプレイヤーがリーダーボード順位と報酬を競う競技期間です。

**シーズン終了時:**
- トッププレイヤーがシーズン報酬を受け取る
- 報酬フラグがリセットされる
- 新しいシーズンが自動的に開始される
- プレイヤー統計（イロレーティング、勝敗）は**保持される**

**デフォルト期間:** 30日

---

## シーズンライフサイクル

```
シーズン開始 → 競争 → シーズン終了 → 報酬 → 新シーズン
    ↓            ↓           ↓           ↓          ↓
  1日目      2-29日目     30日目      31日目    1日目
```

**シーズン中:**
- プレイヤーがランクバトルで競争
- 結果に基づいてイロレーティングが変化
- リーダーボードがリアルタイムで更新

**シーズン終了時:**
- リーダーボードが確定
- シーズン報酬が計算される
- 報酬がGUIで請求可能に
- マイルストーン報酬がリセットされる

**保持:**
- ✅ イロレーティング
- ✅ 総勝敗数
- ✅ 切断回数

**リセット:**
- ❌ マイルストーン報酬フラグ
- ❌ シーズン報酬請求

---

## 設定

`config/cobbleranked/config.json5`:

```json5
{
  "ranked_match": {
    "reset_days": 30  // シーズンの長さ（日数）
  }
}
```

### 一般的な期間

| 期間 | 使用ケース |
|----------|----------|
| 7日 | 週次競技 |
| 14日 | 隔週 |
| **30日** | **月次（推奨）** |
| 60日 | 長期 |
| 90日 | 四半期 |

---

## 自動ローテーション

期間が終了するとシーズンが自動的にローテーション。

**仕組み:**
1. 10分ごとにチェック実行（バトルサーバーのみ）
2. `current_time >= end_time`の場合:
   - 現在のシーズンが終了
   - 新しいシーズンが自動作成
   - プレイヤーにチャットで通知

**設定:** 不要（自動）。

---

## 手動コマンド

### 現在のシーズンを終了

```
/rankedadmin endseason
```

現在のシーズンを即座に強制終了。

### 新しいシーズンを作成

```
/rankedadmin createseason <days> [name]
```

**例:**
```
/rankedadmin createseason 30
/rankedadmin createseason 30 "Summer Championship"
```

### シーズン情報

```
/rankedadmin seasoninfo
```

表示内容:
- 現在のシーズンID
- 開始日
- 終了日  
- 残り日数

## クロスサーバー動作

⚠️ **バトルサーバーシングルトン**

シーズンを管理するのは1つのサーバーのみ（バトルサーバー）。

**設定:**
```json5
{
  "battle_server": ""  // 空 = シーズン管理
}
```

**他のサーバー:**
```json5
{
  "battle_server": "battle1"  // シーズン管理しない
}
```

**理由:** 複数のバトルサーバーは重複したシーズンローテーションを引き起こす。

---

## トラブルシューティング

**シーズンがローテーションしない？**
- バトルサーバーが実行中かチェック
- `reset_days`が設定されているか確認
- サーバーログでエラーをチェック

**複数のシーズンが作成される？**
- バトルサーバーが1つのみ（`battle_server: ""`）か確認
- サーバーログで重複警告をチェック

**プレイヤーが報酬を受け取らない？**
- シーズンが終了したか確認: `/rankedadmin seasoninfo`
- [報酬設定](../configuration/rewards.md)をチェック
