# ランクバトル

---
**CobbleRanked** > **Features** > **ランクバトル**
---

CobbleRankedでのランクバトルの仕組みを学びます。

## 概要

ランクバトルは、プレイヤーがイロレーティングとリーダーボード順位を競う競技的なポケモンバトルです。CobbleRankedはキューから結果までのバトルフロー全体を管理します。

> **[📸 画像必要: バトルフローチャート（キュー参加→マッチング→検証→テレポート→バトル→結果処理→報酬の流れ）]**

## バトルフロー

### 1. キューフェーズ

**プレイヤーがキューに参加:**
1. ランクGUIを開く（`/ranked`）
2. バトル形式を選択（シングルまたはダブルス）
3. キューボタンをクリック
4. マッチメイキングプールに入る

**キュー状態:**
- キュー中: 対戦相手を検索中
- 待機時間が表示される
- いつでもキューを離れることができる

### 2. マッチメイキング

**自動ペアリング:**
- プレイヤーはイロレーティング範囲でマッチング
- 同じバトル形式が必要
- マッチが見つからない場合、時間経過で範囲が拡大

**マッチメイキング基準:**
1. **同じ形式:** 両プレイヤーが同じ形式でキューイング（シングルまたはダブルス）
2. **イロレーティング範囲:** 設定された範囲内（デフォルト±200、時間経過で拡大）
3. **有効なパーティ:** 両プレイヤーが有効なポケモンチームを持っている

**例:**
```
プレイヤーA: 1000イロレーティング、シングルキュー
プレイヤーB: 1050イロレーティング、シングルキュー
結果: マッチ成立！（200イロレーティング範囲内）
```

### 3. バトル前検証

バトル開始前に、両チームが検証されます:

**検証チェック:**
- ✅ チームサイズが要件を満たす（デフォルト: 6）
- ✅ 禁止されたポケモンなし（ブラックリスト）
- ✅ 禁止された技なし
- ✅ 禁止された特性なし
- ✅ 禁止された道具なし
- ✅ 特別な形式要件（有効な場合）

**検証失敗時:**
- プレイヤーがキューから削除される
- 理由付きのエラーメッセージが表示される
- 他のプレイヤーはキューに戻る

### 4. テレポート

**アリーナ選択:**
1. 設定からランダムにアリーナが選択される
2. 両プレイヤーが同時にテレポート
3. 帰還用に前の場所が保存される

**テレポート機能:**
- ディメンション間で動作（オーバーワールド、ネザー、エンド、カスタム）
- インベントリとエフェクトを保持
- 即座のテレポート（ロード画面なし）

### 5. バトル開始

**バトル初期化:**
1. プレイヤーが対面
2. バトルGUIが表示される
3. ポケモンチームがロードされる
4. バトル開始！

**バトル設定:**
- レベル上限が適用される（設定されている場合）
- ターン制限がアクティブ（デフォルト: 100ターン）
- バトル形式が適用される
- 条項が有効化される

### 6. バトルフェーズ

**バトル中:**
- 標準的なCobblemonバトルメカニクス
- ターンベースの戦闘
- 技選択
- 交代メカニクス
- アイテム使用（許可されている場合）

**特殊メカニクス:**
- テラスタル（有効な場合）
- メガシンカ（有効な場合）
- Zワザ（有効な場合）
- ダイマックス（有効な場合）

**バトル追跡:**
- ターンカウンター
- バトル時間
- 形式を記録
- プレイヤーのアクションをログ

### 7. バトル終了

**可能な結果:**

| 結果 | 説明 | イロレーティング変化 |
|---------|-------------|------------|
| **勝利** | すべての相手ポケモンがひんし | 勝者が獲得、敗者が失う |
| **敗北** | すべてのポケモンがひんし | 敗者が失い、勝者が獲得 |
| **降参** | プレイヤーが手動で降参 | 敗北としてカウント |
| **切断** | バトル中にプレイヤーが切断 | 切断回数+1、敗北としてカウント |
| **引き分け** | ターン制限に達した | イロレーティング変化なし |

### 8. 結果処理

**バトル終了後:**

1. **イロレーティング計算:**
   - 勝者がイロレーティングを獲得
   - 敗者がイロレーティングを失う
   - 量はイロレーティングシステム設定に基づく

2. **統計更新:**
   - 勝敗数が増加
   - 勝率が再計算される
   - 切断回数が更新される（切断の場合）
   - 形式固有の統計が更新される

3. **データベース保存:**
   - プレイヤー統計が永続化
   - バトル履歴が記録される
   - リーダーボードが更新される

4. **帰還テレポート:**
   - プレイヤーが前の場所に戻る
   - インベントリが復元される
   - エフェクトが保持される

### 9. 報酬

**バトル後の報酬:**

**マイルストーン報酬**（設定されている場合）:
- 10勝
- 25勝
- 50勝
- 100勝

**通知:**
プレイヤーは以下について通知を受け取ります:
- イロレーティング変化
- 新しいランク順位
- マイルストーン報酬（アンロックされた場合）

## バトル形式

CobbleRankedはシングル、ダブルス、トリプル、マルチ、ランダムバトル形式で独立したランキングをサポートします。

各バトル形式には以下があります:
- **独立したイロレーティング**
- **独立したリーダーボード**
- **形式固有の統計**（勝利、敗北、マッチ）
- **独立したキュー**

**シングルでの勝利はダブルスのイロレーティングに影響しません。**

### 利用可能な形式

#### 競技形式

| 形式 | 説明 | チームサイズ | パーティサイズ | ランク |
|--------|-------------|-----------|------------|--------|
| **SINGLES** | 1v1バトル | 3-6ポケモン | 1プレイヤー | ✅ |
| **DOUBLES** | 2v2バトル | 4-6ポケモン | 1プレイヤー | ✅ |
| **TRIPLES** | 3v3バトル | 5-6ポケモン | 1プレイヤー | ✅ |
| **MULTI** | 2v2チームバトル | 各1-3ポケモン | 2プレイヤー | ✅ |

#### ランダムバトル形式

| 形式 | 説明 | チームサイズ | 世代 | ランク |
|--------|-------------|-----------|------------|--------|
| **RANDOM_SINGLES** | ランダム1v1 | 6ポケモン（自動） | プールベース | ✅ |
| **RANDOM_DOUBLES** | ランダム2v2 | 6ポケモン（自動） | プールベース | ✅ |
| **RANDOM_3V3** | クイックランダム | 3ポケモン（自動） | プールベース | ✅ |

### シングル

伝統的な1v1ポケモンバトル。

**ルール:**
- 3-6ポケモンを連れてくる
- 1v1形式
- バトル前に先発ポケモンを選択

**例:** プレイヤーA vs プレイヤーB（ソロ）

### ダブルス

1プレイヤーが2ポケモンを同時に操作。

**ルール:**
- 4-6ポケモンを連れてくる
- 両方のポケモンが同時に場に出る
- バトル前に2体の先発を選択

**例:** プレイヤーA（2ポケモン）vs プレイヤーB（2ポケモン）

### マルチ（2v2シングル）

チームバトル - 2プレイヤー vs 2プレイヤー。

**ルール:**
- 各プレイヤーが1-3ポケモンを連れてくる
- パートナーは同じパーティにいる必要がある
- 両プレイヤーが先発を選択

**例:** パーティ（プレイヤーA + プレイヤーB）vs パーティ（プレイヤーC + プレイヤーD）

#### マルチのキュー方法

1. パートナーとパーティを組む
2. 両プレイヤーが`/ranked` GUIを開く
3. 「マルチ」形式を選択
4. 両方がキューをクリック

**マッチメイキング:** システムは似た合計イロレーティングの2つのパーティをペアリング。

### 形式選択

プレイヤーはキューイング前に形式を選択:

1. `/ranked` GUIを開く
2. 形式選択をクリック（シングル/ダブルス/マルチ）
3. 「キューに参加」をクリック

### 独立した統計

各形式は別々に追跡:

| 統計 | 形式ごと |
|------|-----------|
| イロレーティング | ✅ |
| 勝利数 | ✅ |
| 敗北数 | ✅ |
| 連勝 | ✅ |
| 総マッチ数 | ✅ |

**例:**
- シングル: 1500イロレーティング、50勝、30敗
- ダブルス: 1000イロレーティング、5勝、5敗
- マルチ: 1200イロレーティング、20勝、15敗

### 形式リーダーボード

各形式には別個のリーダーボードがあります。

**アクセス:**
1. `/ranked` GUI
2. 「リーダーボード」をクリック
3. 形式を選択

**表示:**
- イロレーティング順のトッププレイヤー
- 形式固有のランキング
- シングル/ダブルス/マルチでフィルター

### 形式固有の報酬

シーズンとマイルストーン報酬は形式固有にできます。

**例:**
```json5
{
  "season_rewards": {
    "singles": { /* シングル報酬 */ },
    "doubles": { /* ダブルス報酬 */ }
  }
}
```

プレイヤーは**すべての形式**で報酬を獲得できます。

設定については[報酬システム](../configuration/rewards.md)を参照。

### チームサイズ設定

`config/cobbleranked/config.json5`で形式ごとのチームサイズ制限を設定:

```json5
{
  "ranked_match": {
    "singles": {
      "min_team_size": 3,
      "max_team_size": 6
    },
    "doubles": {
      "min_team_size": 4,
      "max_team_size": 6
    },
    "multi": {
      "min_team_size": 1,
      "max_team_size": 3
    }
  }
}
```

### 形式トラブルシューティング

**マルチでキューできない？**
- パーティが組まれていることを確認（2プレイヤー）
- 両プレイヤーが十分なポケモンを持っている
- 両方が同じ形式を選択したかチェック

**統計が正しく追跡されない？**
- 正しい形式でプレイしているか確認
- リーダーボードで形式をチェック
- 統計は形式独立

**特定の形式でマッチメイキングが遅い？**
- ダブルス/マルチはプレイヤープールが小さい
- [動的マッチメイキング](dynamic-matchmaking.md)を参照

## 切断処理

### 意図的な切断（キレ落ち）

**検出:**
- プレイヤーがバトル中にゲームを閉じる
- クライアントがサーバーから切断
- 降参ボタンが使用されない

**ペナルティ:**
- 切断回数+1
- 敗北としてカウント
- イロレーティングペナルティが適用される
- 対戦相手が自動的に勝利

**切断回数追跡:**
- プレイヤー統計で表示
- 自動的には減少しない
- 管理者がリセット可能: `/rankedadmin flee reset <player>`

### 意図しない切断（クラッシュ/インターネット）

**同じペナルティ:**
- CobbleRankedは意図的と意図しないを区別できない
- すべての切断が同じ扱い
- 悪用を防止

**保護:**
- 管理者が正当なクラッシュのために切断回数を手動でリセット可能
- イロレーティングは手動調整可能: `/rankedadmin elo add <player> <amount>`

### 接続問題

**バトル開始前:**
- プレイヤーがキューから削除される
- ペナルティなし
- キューに再参加可能

**バトル中:**
- 切断として扱われる
- 切断回数が増加
- 対戦相手が勝利

## 引き分け条件

### ターン制限到達

**トリガー:** バトルが最大ターン数に達する（デフォルト: 100）

**結果:**
- バトルが引き分けで終了
- どちらのプレイヤーもイロレーティング変化なし
- 勝敗としてカウントされない
- 切断回数ペナルティなし

**設定:**
```json5
"battle": {
  "maxTurns": 100  // より長いバトルのために増加
}
```

### 両プレイヤーが切断

**稀なシナリオ:** 両プレイヤーが同時に切断

**結果:**
- バトルがキャンセルされる
- イロレーティング変化なし
- 統計更新なし
- ペナルティなし

## 特殊ケース

### バトル中に無効なチーム

**シナリオ:** バトル中にチームが無効になる（例: ハックで取得した禁止ポケモン）

**結果:**
- バトルが即座に終了
- 無効なチームのプレイヤーが敗北
- 対戦相手が勝利
- 重大な警告が発行される

### アリーナの問題

**シナリオ:** バトル中にアリーナワールドがアンロードまたは無効になる

**結果:**
- バトルは継続（バトルインスタンスは別）
- 帰還テレポートはフォールバック場所を使用（スポーン）
- 管理者レビュー用にエラーがログに記録される

### サーバークラッシュ

**シナリオ:** バトル中にサーバーがクラッシュ

**結果:**
- 再起動時にバトルがキャンセルされる
- イロレーティング変化なし
- 統計更新なし
- プレイヤーは前の場所に戻る

## バトル統計

### 形式ごとに追跡

- **総バトル数:** 勝利+敗北
- **勝率:** 勝利 / (勝利 + 敗北) * 100%
- **現在のイロレーティング:** リアルタイムレーティング
- **ピークイロレーティング:** 到達した最高イロレーティング（将来の機能）
- **切断回数:** 切断ペナルティ

### グローバル統計

- **総バトル数:** すべての形式の合計
- **お気に入り形式:** 最もプレイした
- **ベスト形式:** 最高イロレーティング
- **総合勝率:** 合算勝率

## 競技機能

### レベルスケーリング

**レベル上限の強制:**
```json5
"battle": {
  "levelMatch": 70,        // すべてのポケモンがレベル70に
  "forceLevelCap": true    // スケーリングを強制
}
```

**利点:**
- 公平な競争
- 育成アドバンテージなし
- スキルベースのバトル

### 親トレーナー要件

**交換チームを防止:**
```json5
"competitive": {
  "requireOriginalTrainer": true  // 親トレーナーである必要がある
}
```

**効果:**
- プレイヤーはすべてのポケモンの親トレーナーである必要がある
- 完璧なチームの売買/交換を防止
- 正当な捕獲/育成を奨励

### チームサイズ強制

**固定パーティサイズ:**
```json5
"competitive": {
  "teamSize": 6  // 正確に6ポケモンが必要
}
```

**根拠:**
- 標準的な競技形式
- 不公平なアドバンテージを防止
- 公式トーナメントと一貫性

## バトルコマンド

### バトル中

特別なコマンドは不要 - 標準的なCobblemonバトルUIを使用:
- 技をクリックして攻撃
- ポケモンをクリックして交代
- アイテムを使用（許可されている場合）
- 降参ボタンで降参

### バトル前

バトル開始前のコマンド:

```bash
/ranked              # GUIを開く
/queue join singles  # キューに参加
/queue leave         # キューを離れる
```

### バトル後

バトル後のコマンド:

```bash
/stats               # 統計を表示
/leaderboard        # リーダーボードをチェック
/elo                # イロレーティングをチェック
```

## トラブルシューティング

### バトルが開始しない

**症状:** マッチしたがバトルが始まらない

**原因:**
- 無効なポケモンチーム
- ブラックリスト違反
- アリーナ設定エラー

**解決策:**
1. ブラックリスト検証メッセージをチェック
2. アリーナが存在するか確認: `/rankedadmin arena list`
3. コンソールでエラーをチェック

### イロレーティングが更新されない

**症状:** バトル終了後もイロレーティングが変化しない

**原因:**
- 引き分け（ターン制限）
- データベース接続エラー
- イロレーティングシステム設定エラー

**解決策:**
1. バトルが引き分けで終了したかチェック
2. データベース接続を確認
3. コンソールでエラーをチェック

### 帰還テレポート失敗

**症状:** バトル後にプレイヤーがアリーナで動けなくなる

**原因:**
- 前のワールドがアンロードされた
- 座標が無効
- ディメンションエラー

**解決策:**
1. 手動でプレイヤーをテレポート: `/tp <player> <x> <y> <z>`
2. `logs/latest.log`でテレポートエラーをチェック
3. アリーナ設定を確認
