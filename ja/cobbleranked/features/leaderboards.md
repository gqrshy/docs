# リーダーボード

---
**CobbleRanked** > **Features** > **リーダーボード**
---

イロレーティング順のトップランクプレイヤーを表示。

## 機能

- 形式固有（シングル、ダブルスは別々）
- バトル後のリアルタイム更新
- トップ10プレイヤーを表示
- GUIまたはコマンドでアクセス

## リーダーボードの表示

### コマンド経由

```bash
/leaderboard singles
/leaderboard doubles
```

### GUI経由

1. ランクメニューを開く: `/ranked`
2. 「リーダーボード」ボタンをクリック
3. 形式を選択（シングル/ダブルス）

## リーダーボード表示

各プレイヤーについて表示:
- ランク順位（#1、#2など）
- プレイヤー名
- 現在のイロレーティング
- 勝敗記録

例:
```
1. PlayerA - 1847イロレーティング（25勝 - 10敗）
2. PlayerB - 1726イロレーティング（30勝 - 15敗）
3. PlayerC - 1689イロレーティング（28勝 - 14敗）
```

> **[📸 画像必要: リーダーボード表示画面（トップ10プレイヤーのランキング、Elo、勝敗記録が表示されたGUI）]**

## ランキング基準

プレイヤーは以下で順位付け:
1. **イロレーティング**（主要）
2. **勝利数**（タイブレーカー）
3. **参加日**（副次タイブレーカー）

## 設定

**ファイル:** `config/cobbleranked/config.json5`

```json5
{
  "leaderboard": {
    "display_size": 10,          // 表示されるトップNプレイヤー
    "min_battles": 5,            // 表示される最小バトル数
    "update_interval": 60        // 更新頻度（秒）
  }
}
```

## クロスサーバーリーダーボード

クロスサーバーモードでは、リーダーボードはネットワーク全体のすべてのプレイヤーを表示（共有データベース）。

## リーダーボードのリセット

リーダーボードは新しいシーズン開始時に自動的にリセット。手動でリセットするには:

```bash
/rankedarena season create <days> <name>
```

## トラブルシューティング

**リーダーボードが更新されない:**
- キャッシュ更新まで最大60秒待機
- プレイヤーが最小バトル数を持っているかチェック（デフォルト: 5）

**プレイヤーがリーダーボードから欠落:**
- プレイヤーが少なくとも`min_battles`マッチをプレイしたか確認
- プレイヤーのイロレーティングが0より上かチェック
