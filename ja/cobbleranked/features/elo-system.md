# イロレーティングシステム

---
**CobbleRanked** > **Features** > **イロレーティングシステム**
---

マッチメイキングとリーダーボード用のプレイヤースキルレベルを計算。

---

## 概要

イロレーティングは以下を決定:
- **マッチメイキング** - 誰と対戦するか
- **リーダーボード順位** - あなたの順位
- **スキル進行** - 目に見える改善

**開始イロレーティング:** 1000（デフォルト）  
**形式固有:** シングル、ダブルス、マルチは別々のイロレーティングを追跡

---

## イロレーティングの仕組み

- **勝利** → イロレーティング獲得
- **敗北** → イロレーティング損失
- **交換されるポイント**はレーティング差に依存

### 例（Pokemon Showdownモード、K=32）

| シナリオ | あなたのイロレーティング | 相手 | 勝利確率 | 勝利 | 敗北 |
|----------|----------|----------|-----------------|-----|------|
| 互角のマッチ | 1000 | 1000 | 50% | **+16** | -16 |
| アップセット勝利 | 900 | 1200 | 24% | **+24** | -8 |
| 期待勝利 | 1200 | 900 | 76% | **+8** | -24 |
| 大アップセット | 800 | 1400 | 5% | **+30** | -2 |

**重要な洞察:**
- 互角のマッチ（50/50）: ±16イロレーティング
- 上位を倒す = 大きな獲得、小さな損失リスク
- 下位に負ける = 大きな損失、小さな獲得可能性
- システム内の総イロレーティングは一定（ゼロサム）

> **[📸 画像必要: Elo変動の計算例を示す図解（対戦前後のElo変化、期待勝率と実際の結果）]**

## 計算モード

### Pokemon Showdown（推奨）

K因子システムを使用（チェス/Pokemon Showdownに類似）。

```json5
{
  "eloSystem": {
    "mode": "POKEMON_SHOWDOWN",
    "pokemonShowdown": {
      "initialElo": 1000,
      "floorElo": 1000,           // 最小イロレーティング
      "kFactor": 32,               // 標準K因子
      "provisionalMatches": 10,    // 新規プレイヤー期間
      "provisionalKFactor": 64     // 新規プレイヤーには高く
    }
  }
}
```

**K因子:**
- 新規プレイヤー（< 10マッチ）: K = 64（レーティング変化が速い）
- 確立したプレイヤー: K = 32（安定）

> **[📸 画像必要: K因子の違いによるElo変化のグラフ（K=16, 32, 64での変化率比較）]**

### Legacy（シンプル）

最小値と最大値の間のランダムなイロレーティング変化。

```json5
{
  "eloSystem": {
    "mode": "LEGACY",
    "legacy": {
      "minPoints": 10,
      "maxPoints": 30
    }
  }
}
```

勝者が+10から+30イロレーティング獲得（ランダム）。

### Glicko-2（高度）

Chess.comとLichessで使用される最も正確なシステム。レーティングに加えて**レーティング不確実性（RD）**を追跡。

```json5
{
  "eloSystem": {
    "mode": "GLICKO2",
    "glicko2": {
      "initialRating": 1500.0,        // 開始レーティング（標準イロレーティングより高い）
      "initialRD": 350.0,              // レーティング偏差（不確実性）
      "initialVolatility": 0.06,       // レーティング変化の不規則性
      "tau": 0.5,                      // システムボラティリティ制約
      "rdDecayDays": 30                // 非アクティブ時のRD増加
    }
  }
}
```

**仕組み:**
- **レーティング（R）:** スキルレベル（イロレーティングのような）
- **レーティング偏差（RD）:** レーティングへの信頼度
  - 新規プレイヤー: RD = 350（不確実）
  - 20+マッチ後: RD ≈ 50-100（確信）
  - 非アクティブプレイヤー: RD増加（不確実性増加）
- **ボラティリティ（σ）:** 一貫したパフォーマンス

**レーティング変化:**
- 新規プレイヤー（高RD）: マッチあたり**±100ポイント**
- 確立したプレイヤー（低RD）: マッチあたり**±20-40ポイント**
- 不確実なプレイヤーを倒す（高RD）: 小さな獲得
- 確信したプレイヤーを倒す（低RD）: 大きな獲得

> **[📸 画像必要: RD（Rating Deviation）変化の視覚化（試合数に応じたRD減少グラフ、不活動時の増加）]**

**使用タイミング:**
- ✅ 大規模競技サーバー（100+アクティブプレイヤー）
- ✅ 復帰プレイヤーの公平なレーティングを望む
- ✅ レーティング信頼度を考慮する必要がある
- ❌ 小規模カジュアルサーバー（代わりにPokemon Showdownを使用）

## 設定

### K因子ガイド（Pokemon Showdownモード）

K因子はマッチごとのイロレーティング変化量を制御。高い = 速い変化。

**公式:** `イロレーティング変化 = K × (実際の結果 - 期待結果)`

| K因子 | レーティング変化 | 使用ケース | 例 |
|----------|---------------|----------|---------|
| 16 | ±4から±12 | 非常に安定、トップランクのみ | チェスグランドマスター |
| 24 | ±6から±18 | 競技的、遅い進行 | ランク競技 |
| **32** | **±8から±24** | **標準** | **Pokemon Showdownデフォルト** |
| 40 | ±10から±30 | 速い進行、カジュアル | カジュアル競技 |
| 64 | ±16から±48 | 非常に速い（新規プレイヤーのみ） | 最初の10マッチ |

**推奨:**
- **競技サーバー:** K = 32（標準: 64）、仮K = 64
- **カジュアルサーバー:** K = 40（標準: 64）、仮K = 80
- **高活動:** 低いK因子（レーティングが速く安定）
- **低活動:** 高いK因子（レーティングが速く調整）

### イロレーティング減衰

非アクティブプレイヤーがイロレーティングを失い、ラダー停滞を防止。

```json5
{
  "pokemonShowdown": {
    "decay": {
      "enabled": true,
      "runAtUtcHour": 9,
      "slowDecayReduction": 2    // 1日あたり-2イロレーティング
    }
  }
}
```

**例:** 10日間非アクティブ = -20イロレーティング

> **[📸 画像必要: Elo減衰システムの動作例（不活動日数とElo減少の関係グラフ）]**

**注意:** Glicko-2モードはRD減衰を使用（レーティングは維持、不確実性増加）。

## トラブルシューティング

**イロレーティングが変化しない？**
- バトルが完了したかチェック（引き分けではない）
- イロレーティングモードが設定されているか確認
- データベース接続をチェック

**変化が速すぎる/遅すぎる？**
- K因子を調整（`kFactor: 32` → 高く/低く）
- まだ仮設定か確認（最初の10マッチ）

**みんなが1000で動かない？**
- バトルが完了しているか確認
- データベース保存をチェック
- 設定をリロード

## 動的マッチメイキング

プレイヤーがキューで待機する間、イロレーティング範囲を自動的に拡大してより速いマッチング。

### 仕組み

1. **プレイヤーがキューに参加** → 狭いイロレーティング範囲で開始（±200）
2. **マッチが見つからない** → 時間経過で範囲が徐々に拡大
3. **マッチ発見** → バトル開始

**公式:** `range = initial_range + (seconds_waited / expansion_rate)`

### 設定

`config/cobbleranked/config.json5`:

```json5
{
  "matchmaking": {
    "enabled": true,
    "initial_range": 200,           // 開始イロレーティング範囲（±200）
    "expansion_delay": 30,          // 拡大前に30秒待機
    "expansion_rate": 5,            // 5秒ごとに+1イロレーティング
    "max_multiplier": 3.0,          // 最大±600（200 × 3.0）
    "immediate_match_range": 100   // ±100の場合即時マッチ
  }
}
```

#### フィールド

| フィールド | デフォルト | 説明 |
|-------|---------|-------------|
| `enabled` | `true` | 動的拡大を有効化 |
| `initial_range` | `200` | 開始イロレーティング範囲 |
| `expansion_delay` | `30` | 拡大開始前の秒数 |
| `expansion_rate` | `5` | +1イロレーティング拡大あたりの秒数 |
| `max_multiplier` | `3.0` | 最大範囲乗数 |
| `immediate_match_range` | `100` | 即時マッチ閾値 |

### タイムライン例

**1500イロレーティングのプレイヤー:**

| 待機時間 | イロレーティング範囲 | 最小 | 最大 |
|-----------|-----------|-----|-----|
| 0-30秒 | ±200 | 1300 | 1700 |
| 60秒 | ±206 | 1294 | 1706 |
| 120秒 | ±218 | 1282 | 1718 |
| 300秒 | ±254 | 1246 | 1754 |
| 600秒 | ±314 | 1186 | 1814 |
| 2000秒+ | ±600（最大） | 900 | 2100 |

### プリセット

#### 高速（カジュアルサーバー）
```json5
{
  "initial_range": 300,
  "expansion_delay": 15,
  "expansion_rate": 3,
  "max_multiplier": 4.0
}
```

#### バランス（推奨）
```json5
{
  "initial_range": 200,
  "expansion_delay": 30,
  "expansion_rate": 5,
  "max_multiplier": 3.0
}
```

#### 厳格（競技）
```json5
{
  "initial_range": 100,
  "expansion_delay": 60,
  "expansion_rate": 10,
  "max_multiplier": 2.0
}
```

### マッチメイキングトラブルシューティング

**キュー時間が長すぎる？**
- `expansion_delay`を減らす
- `expansion_rate`を減らす
- `initial_range`を増やす

**マッチが不均衡すぎる？**
- `initial_range`を減らす
- `expansion_delay`を増やす
- `max_multiplier`を減らす

**拡大が機能しない？**
- `enabled: true`をチェック
- 設定をリロード: `/rankedadmin reload`
