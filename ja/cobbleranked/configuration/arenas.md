# アリーナ設定

---
**CobbleRanked** > **Configuration** > **アリーナ**
---

ランクマッチのバトルアリーナを設定する方法を学びます。

## ファイルの場所

`config/cobbleranked/arenas.json5`

## 概要

アリーナはランクバトルが行われる物理的な場所です。マッチが見つかると、両プレイヤーはランダムに選択されたアリーナにテレポートされます。

**主な機能:**
- 複数のアリーナをサポート
- ランダムなアリーナ選択
- クロスディメンション対応（オーバーワールド、ネザー、エンド、カスタムディメンション）
- 自動テレポート
- バトル後に前の場所に戻る

## クイックスタート

### 最初のアリーナを作成

1. **ゲーム内でアリーナを建設**（または既存の場所を使用）
2. **プレイヤーを出現させたい地点に立つ**
3. **コマンドを実行:**
   ```
   /rankedadmin arena set main_arena
   ```

これで完了！アリーナが設定されました。

### アリーナを確認

すべての設定されたアリーナをリスト表示:
```
/rankedadmin arena list
```

出力:
```
設定されたアリーナ (1):
  1. main_arena (world: minecraft:overworld, x: 100, y: 64, z: 200)
```

## アリーナ管理

### 複数のアリーナを追加

CobbleRankedはバリエーションのために利用可能なアリーナからランダムに選択:

```bash
/rankedadmin arena set volcano_arena
/rankedadmin arena set ice_arena
/rankedadmin arena set forest_arena
/rankedadmin arena set desert_arena
```

**推奨:** バリエーションのために少なくとも3-5個のアリーナを作成。

### アリーナを削除

アリーナ設定を削除:

```
/rankedadmin arena remove old_arena
```

**警告:** これは物理的な建物を削除せず、ローテーションから削除するだけです。

### アリーナにテレポート

アリーナの場所をプレビュー:

```
/rankedadmin arena tp main_arena
```

以下の用途に便利:
- スポーン地点のテスト
- プレイヤーにアリーナを見せる
- 管理者の高速移動

### すべてのアリーナをリスト

設定されたすべてのアリーナを表示:

```
/rankedadmin arena list
```

出力内容:
- アリーナ名
- ワールド/ディメンション
- 座標（x, y, z）

## ファイル形式

### JSON5 構造

```json5
{
  // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  //  バトルアリーナ
  // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  "arenas": [
    {
      "name": "main_arena",
      "world": "minecraft:overworld",
      "x": 100.5,
      "y": 64.0,
      "z": 200.5,
      "yaw": 180.0,        // プレイヤーの向き（0-360）
      "pitch": 0.0         // プレイヤーの頭の傾き（-90から90）
    },
    {
      "name": "nether_arena",
      "world": "minecraft:the_nether",
      "x": 50.5,
      "y": 80.0,
      "z": -150.5,
      "yaw": 0.0,
      "pitch": 0.0
    }
  ]
}
```

### フィールドリファレンス

| フィールド | 型 | 説明 | 例 |
|-------|------|-------------|---------|
| `name` | 文字列 | 一意のアリーナ識別子 | `"main_arena"` |
| `world` | 文字列 | Minecraftディメンション ID | `"minecraft:overworld"` |
| `x` | 数値 | X座標 | `100.5` |
| `y` | 数値 | Y座標（高さ） | `64.0` |
| `z` | 数値 | Z座標 | `200.5` |
| `yaw` | 数値 | プレイヤーの向き（0-360） | `180.0` |
| `pitch` | 数値 | プレイヤーの頭の傾き（-90から90） | `0.0` |

### ディメンション ID

一般的なディメンション識別子:

| ディメンション | ID |
|-----------|-----|
| オーバーワールド | `minecraft:overworld` |
| ネザー | `minecraft:the_nether` |
| エンド | `minecraft:the_end` |
| カスタム | `modname:dimension_name` |

## アリーナデザインガイドライン

### サイズの推奨

| バトル形式 | 推奨サイズ |
|---------------|------------------|
| シングル（1v1） | 20x20ブロック |
| ダブル（2v2） | 25x25ブロック |

## スポーン地点の配置

### プレイヤーの配置

アリーナを作成すると、プレイヤーは以下の場所にスポーンします:
- 設定で指定された**正確な座標**
- `yaw`値に基づく**向き**
- `pitch`値に基づく**頭の傾き**

### Yaw（向き）

```
     北 (180°)
          |
西 (90°)—+— 東 (-90° または 270°)
          |
     南 (0°)
```

**例:**
- `0.0` - 南向き
- `90.0` - 西向き
- `180.0` - 北向き
- `-90.0` または `270.0` - 東向き

### Pitch（頭の傾き）

```
-90° = 真上を向く
  0° = 真っ直ぐ前を向く
 90° = 真下を向く
```

> **[📸 画像必要: Yaw/Pitch方向の視覚的図解（プレイヤーの向きと角度の関係、コンパス表示）]**

**推奨:** 通常のバトルには`0.0` pitchを使用。

## 高度な設定

### 手動JSON5編集

一括変更のために`arenas.json5`を手動で編集できます:

```json5
{
  "arenas": [
    {
      "name": "arena_1",
      "world": "minecraft:overworld",
      "x": 100.5,
      "y": 64.0,
      "z": 200.5,
      "yaw": 180.0,
      "pitch": 0.0
    },
    // 複数のアリーナ用にコピーして変更
    {
      "name": "arena_2",
      "world": "minecraft:overworld",
      "x": 300.5,
      "y": 64.0,
      "z": 200.5,
      "yaw": 180.0,
      "pitch": 0.0
    }
  ]
}
```

**編集後:**
```
/rankedadmin reload
```

### WorldGuard連携

荒らしからアリーナを保護:

```bash
# リージョンを作成
//pos1
//pos2
/region define main_arena

# ブロック破壊/設置を防止
/region flag main_arena build deny

# PvPを防止（バトルはCobblemonを使用、PvPではない）
/region flag main_arena pvp deny

# Mobスポーンを防止
/region flag main_arena mob-spawning deny
```

### アリーナ選択ロジック

CobbleRankedがアリーナを選択する方法:

1. **すべてのアリーナをロード** `arenas.json5`から
2. **有効なアリーナをフィルター**（ワールドが存在、座標が有効）
3. **ランダム選択** 有効なアリーナから
4. **両プレイヤーをテレポート** 選択されたアリーナに
5. **バトル開始** 即座に

**重み付け:** すべてのアリーナは同じ確率（重み付けシステムなし）。

## 戻りシステム

バトル終了後:

1. **バトル終了**（勝利/敗北/引き分け/切断）
2. **プレイヤーが戻る** 前の場所に
3. **前の場所:** バトル開始時にいた場所
4. **ディメンション保持:** 正しいワールドに戻る

**例:**
- プレイヤーはネザーの`(100, 64, 200)`にいた
- バトルでオーバーワールドアリーナにテレポート
- バトル後、ネザーの`(100, 64, 200)`に戻る

## パフォーマンスの考慮事項

### アリーナの数

- **1-5アリーナ:** パフォーマンスへの影響は無視できる
- **5-20アリーナ:** 目立った影響なし
- **20+アリーナ:** 最小限の影響（アリーナ選択はO(n)ランダム）

**推奨:** 5-10個のアリーナが圧倒せずにバリエーションを提供するのに理想的。

### クロスディメンションアリーナ

ディメンション間のテレポート:
- **チャンクロード:** 短い遅延が発生する可能性（< 1秒）
- **描画距離:** プレイヤーがチャンクのロードを見る可能性
- **TPS影響:** 最新のサーバーでは無視できる

**ベストプラクティス:** チャンクローダーを使用してアリーナチャンクを事前ロード。
