# 用語集

CobbleRankedドキュメントで使用される技術用語と概念。

---

## レーティングシステム

### イロレーティング
マッチ結果に基づいてプレイヤーの強さを計算する数値スキルランキングシステム。CobbleRankedでは、プレイヤーはデフォルトで1000イロからスタートします。

**関連：** [イロシステム](features/elo-system.md)

### K因子
マッチごとのイロ変動量を決定する係数。K因子が高いほど、レーティング変化が速くなります。
- 新規プレイヤー（10試合未満）：K = 64
- 確立されたプレイヤー：K = 32

**関連：** [イロシステム設定](configuration/config.md#elo-system)

### RD（レーティング偏差）
Glicko-2システムにおいて、プレイヤーのレーティングの不確実性を表します。RDが高いほど、レーティングの確実性が低くなります。
- 新規プレイヤー：RD = 350
- 経験豊富なプレイヤー：RD ≈ 50-100
- 非アクティブ時に増加

**関連：** [Glicko-2モード](features/elo-system.md#glicko-2-advanced)

### プロビジョナルマッチ
新規プレイヤーの最初の10試合で、イロがより劇的に変化します（K因子が2倍）。

**関連：** [K因子ガイド](features/elo-system.md#k-factor-guide-pokemon-showdown-mode)

### イロ減衰
非アクティブなプレイヤーのイロを時間経過とともに減少させ、ラダーの停滞を防ぐシステム。
- デフォルト：非アクティブ後1日あたり-2イロ
- UTC 9:00に毎日実行

**関連：** [イロ減衰設定](features/elo-system.md#elo-decay)

## バトルシステム

### バトル条項
公正な競技バトルを保証するために実施されるルール：

#### 種族条項
同じチーム内で重複するポケモンを防止します（デフォルト：有効）。

#### OHKO条項
じわれ、ぜったいれいど等の一撃必殺技を禁止します（デフォルト：有効）。

#### 眠り条項
同時に2匹以上の相手ポケモンを眠らせることを防止します（デフォルト：有効）。

#### 回避率条項
回避率を上げる技（ちいさくなる、かげぶんしん）を禁止します（デフォルト：有効）。

#### アイテム条項
同じチーム内で重複する持ち物を防止します（デフォルト：無効）。

#### ダイマックス条項
ダイマックス/キョダイマックスを禁止します（デフォルト：有効）。

**関連：** [バトルフォーマット](features/ranked-battles.md#battle-clauses)

### OHKO技
通常のダメージ計算を無視する一撃必殺技：
- じわれ
- ぜったいれいど
- つのドリル
- ハサミギロチン

**関連：** [ブラックリスト設定](configuration/blacklist.md)

### バトルフォーマット

#### シングルバトル（1v1）
- チーム：3～6匹のポケモン
- アクティブ：1対1
- 独立したイロランキング

#### ダブルバトル（2v2）
- チーム：4～6匹のポケモン
- アクティブ：2対2
- VGCスタイルフォーマット
- 独立したイロランキング

#### マルチバトル（2v2チーム）
- チーム：プレイヤーごとに1～3匹のポケモン
- アクティブ：2対2
- パーティメンバーとイロを共有

**関連：** [バトルフォーマット](features/ranked-battles.md)

## サーバーシステム

### アリーナ
保存された座標と向きを持つ指定されたバトル場所。マッチ開始時にプレイヤーはアリーナにテレポートします。

**コマンド：** `/rankedadmin arena set <name>`

**関連：** [アリーナ設定](configuration/arenas.md)

### 逃走ペナルティ
バトルを放棄した場合の一時的なマッチメイキング禁止：
- 1～5回逃走：5分間の禁止
- 6～10回逃走：15分間の禁止
- 11回以上逃走：30分間の禁止
- 減衰：24時間ごとに-1回逃走

**関連：** [切断ペナルティ](features/disconnect-penalties.md)

### クロスサーバー
ネットワーク全体でランキングを共有するマルチサーバーアーキテクチャ：
- 必要：MySQL/MongoDB + Redis
- ロビーサーバーとバトルサーバーを分離
- イロをリアルタイムで同期

**関連：** [クロスサーバー設定](advanced/cross-server.md)

### マッチメイキング範囲
対戦相手を見つける際に許容されるイロ差：
- 初期：±200イロ
- 30秒後に拡大
- 最大：±600イロ（3.0倍の乗数）

**関連：** [イロシステム](features/elo-system.md)

## 設定

### JSON5
コメント（`//`と`/* */`）を許可する拡張JSON形式。すべてのCobbleRanked設定ファイルはJSON5を使用します。

**例：**
```json5
{
  // これはコメントです
  "initialElo": 1000,  // 末尾のカンマが許可されます
}
```

**関連：** すべての設定ページ

### ブラックリスト
ランクバトルでポケモン、技、特性、アイテムを制限するシステム。

**カテゴリ：**
- ポケモンラベル（legendary、mythical等）
- 特定のポケモン（ミュウツー、レックウザ）
- 技（バトンタッチ、おはかまいり）
- 特性（ムラっけ、かげふみ）
- アイテム（おうじゃのしるし、きょうせいギブス）

**関連：** [ブラックリスト設定](configuration/blacklist.md)

### ラベル
ブラックリスト用のポケモンカテゴリタグ：
- `legendary` - 伝説のポケモン
- `mythical` - 幻のポケモン
- `restricted` - VGC制限ポケモン
- `ultra_beast` - ウルトラビースト
- `paradox` - パラドックスポケモン
- `sub_legendary` - 準伝説ポケモン

**関連：** [ブラックリストラベル](configuration/blacklist.md#label-categories)

### 制限ラベル制限
完全禁止ではなく、限定的な使用を許可するVGCスタイルの制限。

**例：** チームごとに最大2匹の制限伝説ポケモンを許可
```json5
{
  "restricted_label_limits": {
    "restricted": 2
  }
}
```

**関連：** [ラベル制限](configuration/blacklist.md#label-limits-vgc-style)

## シーズンシステム

### シーズン
時間制限のある競技期間（デフォルト：30日）。期間終了後にランキングがリセットされます。

**機能：**
- 自動ローテーション
- トップ3プレイヤー報酬
- イロが初期値にリセット
- 勝敗統計のリセット

**関連：** [シーズン](features/seasons.md)

### シーズン報酬
シーズン終了時にトップランクのプレイヤーに与えられる報酬。フォーマットごと（シングル/ダブル/マルチ）に設定されます。

**例：**
```json5
{
  "rewards": {
    "1": ["give %player% diamond 64"],
    "2": ["give %player% emerald 48"],
    "3": ["give %player% gold_ingot 32"]
  }
}
```

**関連：** [報酬設定](configuration/rewards.md)

### マイルストーン報酬
勝利数またはイロマイルストーンに到達した際のアチーブメントベースの報酬。

**例：**
```json5
{
  "win_milestones": {
    "10": ["give %player% rare_candy 5"],
    "50": ["give %player% master_ball 1"]
  }
}
```

**関連：** [マイルストーン報酬](configuration/rewards.md#milestone-rewards)

## 技術用語

### Redis
リアルタイムのクロスサーバー通信とキュー同期に使用されるインメモリデータストア。

**必要条件：** クロスサーバー設定

**関連：** [データベース設定](advanced/database.md#redis-configuration)

### SQLite
デフォルトの埋め込みデータベース。プレイヤー統計、イロ、マッチ履歴をローカルに保存します。

**制限：** シングルサーバーのみ（クロスサーバー不可）

**関連：** [データベース設定](advanced/database.md)

### MySQL
クロスサーバー設定用のリレーショナルデータベース（2～5サーバー推奨）。

**関連：** [データベース設定](advanced/database.md#mysql)

### MongoDB
クロスサーバー設定用のNoSQLデータベース（5台以上のサーバー、クラウド対応）。

**関連：** [データベース設定](advanced/database.md#mongodb)

### Velocity
ネットワーク内の複数のサーバーを接続するための最新のMinecraftプロキシ。

**必要条件：** クロスサーバーランクバトル

**関連：** [クロスサーバー設定](advanced/cross-server.md)

## プレイヤーシステム

### キュー
イロとフォーマットの好みに基づいてプレイヤーをマッチングするマッチメイキングシステム。

**動作：**
- イロ範囲内で検索
- 30秒ごとに範囲を拡大
- 最大範囲に達した後に自動キャンセル

**関連：** [ランクバトル](features/ranked-battles.md)

### リーダーボード
フォーマットごとのリアルタイムトップ10ランキング。各マッチ後に更新されます。

**表示：** `/ranked` GUIまたはゲーム内コマンド

**関連：** [リーダーボード](features/leaderboards.md)

### ターンタイマー
遅延行為を防ぐためのターンごとのオプションの制限時間。

**デフォルト：** 無効（制限時間なし）

**関連：** [ターンタイマー](features/turn-timer.md)
