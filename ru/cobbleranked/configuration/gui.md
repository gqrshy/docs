# Настройка GUI и языков

---
**CobbleRanked** > **Конфигурация** > **GUI и языки**
---

Настройте внешний вид интерфейса CobbleRanked, расположение элементов и сообщения.

## Файлы GUI

**Расположение:** `config/cobbleranked/gui/`

CobbleRanked использует файлы GUI для конкретных языков:

```
gui/
├── gui-enUs.json5    # Английский
├── gui-jaJp.json5    # Японский
├── gui-ptBr.json5    # Португальский
└── gui-ruRu.json5    # Русский
```

Каждый файл содержит **все** определения GUI для этого языка.

## Доступные GUI

- **gui_ranked** - Главное меню (очередь, статистика, таблицы лидеров)
- **gui_top_ranked** - Отображение таблицы лидеров
- **gui_prepare_combat** - Предварительный просмотр команды перед боем
- **gui_rewards** - Награды за сезоны/достижения
- **gui_ranked_blocked** - Просмотр черного списка
- **closeGui** - Кнопка закрытия (общая)

## Базовая настройка

### Изменение отображения предмета

```json5
{
  "gui_ranked": {
    "title": "Рейтинговые бои",
    "rows": 3,
    "items": {
      "queue_singles": {
        "item": "minecraft:iron_sword",  // Изменить предмет
        "name": "Очередь одиночные",     // Изменить название
        "slot": 11,                       // Изменить позицию
        "lore": [
          "Нажмите, чтобы присоединиться к очереди одиночных боёв"
        ]
      }
    }
  }
}
```

### Поля предмета

| Поле | Описание | Обязательное |
|-------|-------------|----------|
| `item` | ID предмета (например, `minecraft:diamond`) | Да |
| `name` | Отображаемое название (поддерживает цветовые коды) | Да |
| `slot` | Слот инвентаря (0-53) | Да |
| `lore` | Массив строк описания | Нет |
| `amount` | Размер стака предмета (1-64) | Нет |
| `glow` | Добавить свечение зачарования | Нет |

### Цветовые коды

Используйте `&` для цветов:
- `&a` - Зелёный
- `&c` - Красный
- `&e` - Жёлтый
- `&b` - Голубой
- `&6` - Золотой
- `&l` - Жирный
- `&r` - Сброс

Пример:
```json5
"name": "&a&lОчередь одиночные"  // Жирный зелёный текст
```

## Плейсхолдеры

Используйте плейсхолдеры для динамического контента:

| Плейсхолдер | Описание |
|-------------|-------------|
| `{elo}` | Рейтинг Elo игрока |
| `{wins}` | Количество побед |
| `{losses}` | Количество поражений |
| `{rank}` | Позиция игрока в рейтинге |
| `{tier}` | Уровень игрока (Pokeball, Greatball и т.д.) |
| `{season}` | Название текущего сезона |
| `{queue_count}` | Игроки в очереди |

Пример:
```json5
"lore": [
  "Ваш Elo: &e{elo}",
  "Победы: &a{wins} &7| Поражения: &c{losses}"
]
```

> **[📸 ТРЕБУЕТСЯ ИЗОБРАЖЕНИЕ: Пример предмета GUI с применёнными плейсхолдерами (подсказка с реальными значениями Elo и побед/поражений)]**

## Применение изменений

1. Отредактируйте файл GUI
2. Выполните `/rankedadmin reload`
3. Переоткройте GUI, чтобы увидеть изменения

> **[📸 ТРЕБУЕТСЯ ИЗОБРАЖЕНИЕ: Сравнение GUI до и после настройки (различия в предметах/расположении по умолчанию и пользовательских)]**

**Примечание:** Перезагрузка конфигурации автоматически определяет язык из `config.json5`.

## Устранение неполадок

**GUI не обновляется:**
- Проверьте синтаксис JSON5 (используйте [JSONLint](https://jsonlint.com/))
- Убедитесь, что кодировка файла UTF-8
- Выполните `/rankedadmin reload`

**Предметы не отображаются:**
- Проверьте номера слотов (диапазон 0-53)
- Проверьте, что ID предметов валидны

**Цвета не работают:**
- Используйте `&`, а не `§` для цветовых кодов
- Убедитесь, что используете `&r` для сброса форматирования

## Настройка языка

### Обзор

CobbleRanked включает 4 встроенных языка:
- **Английский** (`en-Us.json5`) - По умолчанию
- **Японский** (`ja-Jp.json5`)
- **Португальский** (`pt-Br.json5`)
- **Русский** (`ru-Ru.json5`)

**Файлы:** `config/cobbleranked/language/`

Все сообщения, уведомления и тексты полностью настраиваются через языковые файлы JSON5.

### Выбор языка

Отредактируйте `config/cobbleranked/config.json5`:

```json5
{
  "language": "en-Us"  // Измените на: en-Us, ja-Jp, pt-Br, ru-Ru
}
```

Затем перезагрузите:
```bash
/rankedadmin reload
```

## Структура языкового файла

### Формат файла

Языковые файлы используют формат JSON5 с парами ключ-значение:

```json5
{
  "message_key": "Содержание сообщения с {плейсхолдером}",
  "another_key": "Другое сообщение",
  "color_example": "&aЗелёный &cКрасный &eЖёлтый текст"
}
```

### Пример: en-Us.json5

```json5
{
  // Сообщения очереди
  "joined-queue": "&aВы присоединились к очереди рейтингового матча.",
  "left-queue": "&cВы покинули очередь рейтингового матча.",
  "remaingFila": "&cПоиск матча... &7(&e{remaining}&7) &7(Позиция: &e{posicao}&7)",

  // Сообщения боя
  "match-winner": "&aПобеда!!",
  "match-winner-subtitle": "&aВы получили &f{gain} &aочков ELO.",
  "match-loser": "&cПоражение!!",
  "match-loser-subtitle": "&cВы потеряли &f{lose} &aочков ELO.",

  // Сообщения валидации
  "pokemon-blacklist": "&cВы используете покемона, который находится в чёрном списке.",
  "no-pokemon-ability": "&cУ вашего покемона способность, которая не разрешена в рейтинговых боях."
}
```

## Категории сообщений

### Сообщения очереди

| Ключ | Описание | Пример |
|-----|-------------|---------|
| `joined-queue` | Уведомление о присоединении к очереди | "Вы присоединились к очереди рейтингового матча." |
| `left-queue` | Уведомление о выходе из очереди | "Вы покинули очередь рейтингового матча." |
| `remaingFila` | Статус ожидания в очереди | "Поиск матча... (30s) (Позиция: 1)" |
| `match-found` | Заголовок о найденном матче | "Матч найден!" |
| `prepare-queue-cancel` | Сообщение об отмене матча | "Матч был отменён. (Игрок отключился)" |

### Сообщения боя

| Ключ | Описание | Пример |
|-----|-------------|---------|
| `battle-start-title` | Заголовок начала боя | "Начало боя!" |
| `battle-countdown` | Номер обратного отсчёта | "3" |
| `match-started` | Уведомление о начале матча | "Матч начался" |
| `match-winner` | Заголовок победы | "Победа!!" |
| `match-winner-subtitle` | Подзаголовок победы с приростом Elo | "Вы получили 25 очков ELO." |
| `match-loser` | Заголовок поражения | "Поражение!!" |
| `match-loser-subtitle` | Подзаголовок поражения с потерей Elo | "Вы потеряли 18 очков ELO." |

### Сообщения валидации

| Ключ | Описание | Пример |
|-----|-------------|---------|
| `pokemon-blacklist` | Ошибка запрещённого покемона | "Вы используете покемона, который находится в чёрном списке." |
| `no-pokemon-ability` | Ошибка запрещённой способности | "У вашего покемона способность, которая не разрешена в рейтинговых боях." |
| `blocked-item-move` | Ошибка запрещённого приёма | "У вас есть покемон с приёмами, которые запрещены в рейтинговых боях." |
| `no-pokemon-item` | Ошибка запрещённого предмета | "У вашего покемона экипирован предмет, который не разрешён в рейтинговых боях." |
| `limit-pokemon` | Ошибка размера команды | "Вам нужно иметь 6 покемонов, чтобы присоединиться к рейтинговому матчу." |

### Админские сообщения

| Ключ | Описание | Пример |
|-----|-------------|---------|
| `ranked-reload-adm` | Успешная перезагрузка конфигурации | "Настройки успешно перезагружены!" |
| `set-elo-adm` | Команда установки Elo | "Elo 1500 успешно установлен для игрока Steve" |
| `arena-not-found` | Ошибка "арена не найдена" | "Арена volcano_arena не найдена" |
| `noPermCommand` | Отказ в доступе | "У вас нет прав для использования этой команды." |

## Плейсхолдеры сообщений

Многие сообщения поддерживают плейсхолдеры, которые динамически заменяются:

### Плейсхолдеры игрока и матча

| Плейсхолдер | Описание | Пример использования |
|-------------|-------------|---------------|
| `{player}` | Имя игрока | "match-finished": "{player} победил!" |
| `{player1}` | Имя первого игрока | "ranked-started": "{player1} против {player2}" |
| `{player2}` | Имя второго игрока | "prepare-queue-subtitle": "{player1} против {player2}" |
| `{winner}` | Имя победителя | "match-finished": "{winner} победил!" |
| `{loser}` | Имя проигравшего | "match-finished": "{winner} победил {loser}!" |

### Плейсхолдеры Elo и статистики

| Плейсхолдер | Описание | Пример использования |
|-------------|-------------|---------------|
| `{gain}` | Полученный Elo | "match-winner-subtitle": "Вы получили {gain} ELO" |
| `{lose}` | Потерянный Elo | "match-loser-subtitle": "Вы потеряли {lose} ELO" |
| `{elo}` | Рейтинг Elo | "elo-up": "Вы повысились до {elo}." |
| `{wins}` | Количество побед | "Ваши победы: {wins}" |
| `{losses}` | Количество поражений | "Ваши поражения: {losses}" |

### Плейсхолдеры очереди и системы

| Плейсхолдер | Описание | Пример использования |
|-------------|-------------|---------------|
| `{remaining}` | Оставшееся время | "remaingFila": "Поиск... ({remaining})" |
| `{posicao}` | Позиция в очереди | "remaingFila": "Позиция: {posicao}" |
| `{time}` | Значение времени | "match-started-subtitle": "Длительность {time}m" |
| `{limit}` | Лимит размера команды | "limit-pokemon": "Вам нужно {limit} покемонов" |
| `{arena}` | Название арены | "arena-not-found": "Арена {arena} не найдена" |

### Плейсхолдеры покемонов

| Плейсхолдер | Описание | Пример использования |
|-------------|-------------|---------------|
| `{pokemon}` | Название покемона | "pokemon_switched": "Выбран {pokemon}" |
| `{level}` | Уровень покемона | "Уровень: {level}" |
| `{current}` | Текущее HP | "HP: {current}/{max}" |
| `{max}` | Максимальное HP | "HP: {current}/{max}" |
| `{label}` | Категория покемона | "team_selection_label_limit_item": "{label}: {current}/{limit}" |

### Пример с плейсхолдерами

```json5
{
  "match-finished": "&8* &f{winner} &cтолько что выиграл рейтинговый матч против &f{loser}.",
  "elo-up": "&aВы повысились до &f{elo}.",
  "remaingFila": "&cПоиск матча... &7(&e{remaining}&7) &7(Позиция: &e{posicao}&7)"
}
```

**Результат:**
- `{winner}` → "Steve"
- `{loser}` → "Alex"
- Вывод: "* Steve только что выиграл рейтинговый матч против Alex."

## Устранение неполадок с языком

### Язык не загружается

**Симптомы:** Всё ещё показывает английский после смены языка

**Решения:**
1. Проверьте настройку `language` в `config.json5`
2. Убедитесь, что имя файла точно совпадает (с учётом регистра)
3. Перезагрузите: `/rankedadmin reload`
4. Проверьте консоль на наличие ошибок

### Отсутствующие сообщения

**Симптомы:** Некоторые сообщения показывают ключ вместо текста

**Решения:**
1. Убедитесь, что все ключи из `en-Us.json5` существуют в вашем файле
2. Скопируйте отсутствующие ключи из английского файла
3. Перезагрузите конфигурацию

### Плейсхолдеры не заменяются

**Симптомы:** Сообщение показывает `{player}` вместо имени игрока

**Решения:**
1. Проверьте написание плейсхолдера (с учётом регистра)
2. Убедитесь, что плейсхолдер используется в правильном сообщении
3. Используйте правильный плейсхолдер для типа сообщения

## Лучшие практики

### Языковые файлы

**✅ ДЕЛАЙТЕ:**
- Синхронизируйте все языковые файлы (одинаковые ключи)
- Тестируйте сообщения в игре после изменений
- Используйте последовательную цветовую схему для сообщений
- Сохраняйте форматирование плейсхолдеров
- Используйте комментарии для организации

**❌ НЕ ДЕЛАЙТЕ:**
- Не удаляйте и не переименовывайте ключи (вызовет ошибки)
- Не забывайте экранировать специальные символы
- Не используйте несуществующие имена плейсхолдеров
- Не смешивайте разные стили кавычек непоследовательно

### Специальные символы

#### Апострофы

Используйте `''` (двойной апостроф) для экранирования одинарных кавычек:

```json5
{
  "lead_selection_gui_opponent_pokemon": "&7Покемон противника"
}
```

#### Обратные слэши

Используйте `\\` для экранирования обратных слэшей:

```json5
{
  "permission-message": "&cУ вас нет разрешения"
}
```
