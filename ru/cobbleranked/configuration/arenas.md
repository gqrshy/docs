# Конфигурация арен

---
**CobbleRanked** > **Конфигурация** > **Арены**
---

Узнайте, как настроить боевые арены для рейтинговых матчей.

## Расположение файла

`config/cobbleranked/arenas.json5`

## Обзор

Арены - это физические локации, где проходят рейтинговые бои. Когда найден матч, оба игрока телепортируются на случайно выбранную арену.

**Ключевые особенности:**
- Поддержка нескольких арен
- Случайный выбор арены
- Поддержка кросс-измерений (Обычный мир, Нижний мир, Край, пользовательские измерения)
- Автоматическая телепортация
- Возврат на предыдущую локацию после боя

## Быстрый старт

### Создайте свою первую арену

1. **Постройте арену** в игре (или используйте существующую локацию)
2. **Встаньте в точке спавна**, где хотите, чтобы появлялись игроки
3. **Выполните команду:**
   ```
   /rankedadmin arena set main_arena
   ```

Вот и всё! Ваша арена теперь настроена.

### Проверка арены

Список всех настроенных арен:
```
/rankedadmin arena list
```

Вывод:
```
Настроенные арены (1):
  1. main_arena (мир: minecraft:overworld, x: 100, y: 64, z: 200)
```

## Управление аренами

### Добавление нескольких арен

CobbleRanked случайным образом выбирает из доступных арен для разнообразия:

```bash
/rankedadmin arena set volcano_arena
/rankedadmin arena set ice_arena
/rankedadmin arena set forest_arena
/rankedadmin arena set desert_arena
```

**Рекомендация:** Создайте как минимум 3-5 арен для разнообразия.

### Удаление арены

Удалить конфигурацию арены:

```
/rankedadmin arena remove old_arena
```

**Предупреждение:** Это не удаляет физическую постройку, только убирает её из ротации.

### Телепортация на арену

Предварительный просмотр локации арены:

```
/rankedadmin arena tp main_arena
```

Полезно для:
- Тестирования точек спавна
- Показа арен игрокам
- Быстрого перемещения для админов

### Список всех арен

Просмотр всех настроенных арен:

```
/rankedadmin arena list
```

Вывод показывает:
- Название арены
- Мир/измерение
- Координаты (x, y, z)

## Формат файла

### Структура JSON5

```json5
{
  // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  //  БОЕВЫЕ АРЕНЫ
  // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  "arenas": [
    {
      "name": "main_arena",
      "world": "minecraft:overworld",
      "x": 100.5,
      "y": 64.0,
      "z": 200.5,
      "yaw": 180.0,        // Направление взгляда игрока (0-360)
      "pitch": 0.0         // Наклон головы игрока (-90 to 90)
    },
    {
      "name": "nether_arena",
      "world": "minecraft:the_nether",
      "x": 50.5,
      "y": 80.0,
      "z": -150.5,
      "yaw": 0.0,
      "pitch": 0.0
    }
  ]
}
```

### Справочник полей

| Поле | Тип | Описание | Пример |
|-------|------|-------------|---------|
| `name` | String | Уникальный идентификатор арены | `"main_arena"` |
| `world` | String | ID измерения Minecraft | `"minecraft:overworld"` |
| `x` | Number | Координата X | `100.5` |
| `y` | Number | Координата Y (высота) | `64.0` |
| `z` | Number | Координата Z | `200.5` |
| `yaw` | Number | Направление взгляда игрока (0-360) | `180.0` |
| `pitch` | Number | Наклон головы игрока (-90 to 90) | `0.0` |

### ID измерений

Распространённые идентификаторы измерений:

| Измерение | ID |
|-----------|-----|
| Обычный мир | `minecraft:overworld` |
| Нижний мир | `minecraft:the_nether` |
| Край | `minecraft:the_end` |
| Пользовательское | `modname:dimension_name` |

## Позиционирование точки спавна

### Размещение игроков

При создании арены игроки появляются:
- **Точные координаты**, указанные в конфигурации
- **Направление взгляда** на основе значения `yaw`
- **Наклон головы** на основе значения `pitch`

### Yaw (направление взгляда)

```
     Север (180°)
          |
Запад (90°)—+— Восток (-90° or 270°)
          |
     Юг (0°)
```

**Примеры:**
- `0.0` - Взгляд на юг
- `90.0` - Взгляд на запад
- `180.0` - Взгляд на север
- `-90.0` или `270.0` - Взгляд на восток

### Pitch (наклон головы)

```
-90° = Взгляд прямо вверх
  0° = Взгляд прямо вперёд
 90° = Взгляд прямо вниз
```

> **[📸 ТРЕБУЕТСЯ ИЗОБРАЖЕНИЕ: Визуальная диаграмма направлений Yaw/Pitch (связь между направлением игрока и углами, отображение компаса)]**

**Рекомендация:** Используйте `0.0` pitch для обычных боёв.

## Продвинутая конфигурация

### Ручное редактирование JSON5

Вы можете вручную редактировать `arenas.json5` для массовых изменений:

```json5
{
  "arenas": [
    {
      "name": "arena_1",
      "world": "minecraft:overworld",
      "x": 100.5,
      "y": 64.0,
      "z": 200.5,
      "yaw": 180.0,
      "pitch": 0.0
    },
    // Копируйте и изменяйте для нескольких арен
    {
      "name": "arena_2",
      "world": "minecraft:overworld",
      "x": 300.5,
      "y": 64.0,
      "z": 200.5,
      "yaw": 180.0,
      "pitch": 0.0
    }
  ]
}
```

**После редактирования:**
```
/rankedadmin reload
```

### Интеграция с WorldGuard

Защитите арены от гриферства:

```bash
# Создать регион
//pos1
//pos2
/region define main_arena

# Предотвратить ломание/размещение блоков
/region flag main_arena build deny

# Предотвратить PvP (бой использует Cobblemon, а не PvP)
/region flag main_arena pvp deny

# Предотвратить спавн мобов
/region flag main_arena mob-spawning deny
```

### Логика выбора арены

Как CobbleRanked выбирает арены:

1. **Загрузить все арены** из `arenas.json5`
2. **Отфильтровать валидные арены** (мир существует, координаты валидны)
3. **Случайный выбор** из валидных арен
4. **Телепортировать обоих игроков** на выбранную арену
5. **Начать бой** немедленно

**Взвешивание:** Все арены имеют равную вероятность (нет системы взвешивания).

## Система возврата

После окончания боя:

1. **Бой завершается** (победа/поражение/ничья/побег)
2. **Игроки возвращаются** на свою предыдущую локацию
3. **Предыдущая локация:** Где они были, когда начался бой
4. **Измерение сохраняется:** Возврат в правильный мир

**Пример:**
- Игрок был в Нижнем мире на `(100, 64, 200)`
- Бой телепортирует на арену в Обычном мире
- После боя возвращается в Нижний мир `(100, 64, 200)`

## Соображения производительности

### Количество арен

- **1-5 арен:** Незначительное влияние на производительность
- **5-20 арен:** Нет заметного влияния
- **20+ арен:** Минимальное влияние (выбор арены - O(n) случайный)

**Рекомендация:** 5-10 арен идеально для разнообразия без перегрузки.

### Кросс-измерения арен

Телепортация между измерениями:
- **Загрузка чанков:** Может вызвать краткую задержку (< 1 секунды)
- **Дистанция отрисовки:** Игроки могут видеть загружающиеся чанки
- **Влияние на TPS:** Незначительное на современных серверах

**Лучшая практика:** Предварительно загружайте чанки арен с помощью загрузчиков чанков.
