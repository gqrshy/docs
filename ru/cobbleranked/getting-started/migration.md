# Руководство по миграции

---
**CobbleRanked** > **Начало работы** > **Миграция**
---

Обновление CobbleRanked до последней версии.

## Перед миграцией

### 1. Резервное копирование всего

```bash
# Папка конфигурации
cp -r config/cobbleranked config/cobbleranked.backup

# База данных SQLite
cp config/cobbleranked/ranked.db config/cobbleranked/ranked.db.backup

# База данных MySQL
mysqldump -u ranked -p cobbleranked > cobbleranked_backup.sql
```

### 2. Остановка сервера

```bash
/stop
```

## Шаги миграции

### 1. Обновление файла мода

```bash
# Удалите старую версию
rm mods/CobbleRanked-*.jar

# Добавьте новую версию
cp CobbleRanked-1.0.0.jar mods/
```

### 2. Запуск сервера

CobbleRanked автоматически:
- Мигрирует схему базы данных
- Конвертирует старые конфигурации в JSON5
- Создает отсутствующие конфигурационные файлы

### 3. Проверка миграции

```bash
# Проверьте логи на наличие ошибок
tail -n 100 logs/latest.log | grep CobbleRanked

# Тест в игре
/ranked
```

## Изменения для конкретных версий

### Миграция на 1.0.0+

**База данных:**
- Автоматическая миграция с одного Эло на статистику по форматам
- Старая таблица `player_ranked_stats` → таблица `format_stats`

**Конфигурация:**
- YAML удален, только JSON5
- Новые файлы: `restrictions.json5`, `inventory.json5`

**Что делать:**
1. Обновите файл мода
2. Запустите сервер (автоматическая миграция)
3. Проверьте, работает ли GUI `/ranked`
4. Проверьте сохранность статистики игроков

### Настройка кросс-сервера (чистая установка)

Если добавляете режим кросс-сервера:

1. Установите MySQL/MongoDB
2. Установите Redis
3. Настройте все серверы:

```json5
{
  "cross_server": {
    "enabled": true,
    "server_id": "main1",  // или "battle", "lobby"
    "battle_server": "battle"  // или "" для сервера сражений
  }
}
```

См. [Настройка кросс-сервера](../advanced/cross-server.md)

## Устранение неполадок

**Миграция не удалась:**
- Проверьте логи: `logs/latest.log`
- Восстановите резервную копию и попробуйте снова
- Сообщите о проблеме в Discord с фрагментом лога

**Потеряна статистика игроков:**
- Проверьте наличие резервной копии базы данных
- Проверьте таблицу `format_stats` в базе данных
- Используйте резервную копию для восстановления при необходимости

**Конфигурации не конвертируются:**
- Удалите `config/cobbleranked/` (после резервного копирования!)
- Перезапустите сервер для генерации значений по умолчанию
- Повторно примените пользовательские настройки

**Кросс-сервер не работает:**
- Проверьте, запущены ли MySQL/Redis
- Проверьте, что все серверы имеют одинаковую конфигурацию базы данных
- Проверьте соединение: `redis-cli ping`

## Откат (при необходимости)

```bash
# Остановите сервер
/stop

# Восстановите старый мод
rm mods/CobbleRanked-*.jar
cp backup/CobbleRanked-old.jar mods/

# Восстановите конфигурации
rm -rf config/cobbleranked
cp -r config/cobbleranked.backup config/cobbleranked

# Восстановите базу данных (SQLite)
cp config/cobbleranked/ranked.db.backup config/cobbleranked/ranked.db

# Восстановите базу данных (MySQL)
mysql -u ranked -p cobbleranked < cobbleranked_backup.sql

# Запустите сервер
```
