# Рейтинговые бои

---
**CobbleRanked** > **Возможности** > **Рейтинговые бои**
---

Узнайте, как работают рейтинговые бои в CobbleRanked.

## Обзор

Рейтинговые бои - это конкурентные бои покемонов, где игроки соревнуются за рейтинг Elo и позиции в таблице лидеров. CobbleRanked управляет всем процессом боя от очереди до результатов.

> **[📸 ТРЕБУЕТСЯ ИЗОБРАЖЕНИЕ: Блок-схема боя (присоединение к очереди→матчмейкинг→валидация→телепортация→бой→обработка результатов→награды)]**

## Процесс боя

### 1. Фаза очереди

**Игрок присоединяется к очереди:**
1. Открывает рейтинговый GUI (`/ranked`)
2. Выбирает формат боя (Одиночные или Парные)
3. Нажимает кнопку очереди
4. Входит в пул матчмейкинга

**Статус очереди:**
- В очереди: Поиск противника
- Отображается время ожидания
- Можно выйти из очереди в любой момент

### 2. Матчмейкинг

**Автоматическое создание пар:**
- Игроки сопоставляются по диапазону Elo
- Требуется одинаковый формат боя
- Диапазон расширяется со временем, если матч не найден

**Критерии матчмейкинга:**
1. **Одинаковый формат:** Оба игрока в очереди для одного формата (Одиночные или Парные)
2. **Диапазон Elo:** В пределах настроенного диапазона (по умолчанию ±200, расширяется со временем)
3. **Валидная команда:** У обоих игроков валидные команды покемонов

**Пример:**
```
Игрок A: 1000 Elo, очередь одиночных
Игрок B: 1050 Elo, очередь одиночных
Результат: Матч найден! (в пределах диапазона 200 Elo)
```

### 3. Предварительная валидация

Перед началом боя обе команды проверяются:

**Проверки валидации:**
- ✅ Размер команды соответствует требованиям (по умолчанию: 6)
- ✅ Нет запрещённых покемонов (черный список)
- ✅ Нет запрещённых приёмов
- ✅ Нет запрещённых способностей
- ✅ Нет запрещённых предметов
- ✅ Требования специального формата (если включены)

**Если валидация не пройдена:**
- Игрок удаляется из очереди
- Отображается сообщение об ошибке с причиной
- Другой игрок возвращается в очередь

### 4. Телепортация

**Выбор арены:**
1. Случайным образом выбирается арена из конфигурации
2. Оба игрока телепортируются одновременно
3. Предыдущая локация сохраняется для возврата

**Возможности телепортации:**
- Работает между измерениями (Обычный мир, Нижний мир, Край, пользовательские)
- Сохраняет инвентарь и эффекты
- Мгновенная телепортация (без экрана загрузки)

### 5. Начало боя

**Инициализация боя:**
1. Игроки стоят лицом друг к другу
2. Появляется GUI боя
3. Команды покемонов загружены
4. Бой начинается!

**Настройки боя:**
- Принудительный лимит уровня (если настроено)
- Активен лимит ходов (по умолчанию: 100 ходов)
- Применён формат боя
- Включены клаузы

### 6. Фаза боя

**Во время боя:**
- Стандартная механика боя Cobblemon
- Пошаговый бой
- Выбор приёмов
- Механика замены
- Использование предметов (если разрешено)

**Специальные механики:**
- Terastallization (если включена)
- Mega Evolution (если включена)
- Z-Moves (если включены)
- Dynamax (если включён)

**Отслеживание боя:**
- Счётчик ходов
- Время боя
- Записанный формат
- Действия игроков залогированы

### 7. Окончание боя

**Возможные исходы:**

| Исход | Описание | Изменение Elo |
|---------|-------------|------------|
| **Победа** | Все покемоны противника побеждены | Победитель получает, проигравший теряет |
| **Поражение** | Все ваши покемоны побеждены | Проигравший теряет, победитель получает |
| **Сдача** | Игрок вручную сдался | Считается как поражение |
| **Отключение** | Игрок отключился во время боя | Счётчик побегов +1, считается как поражение |
| **Ничья** | Достигнут лимит ходов | Без изменения Elo |

### 8. Обработка результатов

**После окончания боя:**

1. **Расчёт Elo:**
   - Победитель получает Elo
   - Проигравший теряет Elo
   - Количество основано на конфигурации системы Elo

2. **Обновление статистики:**
   - Увеличиваются победы/поражения
   - Пересчитывается процент побед
   - Обновляется счётчик побегов (если отключение)
   - Обновляется статистика для конкретного формата

3. **Сохранение в базе данных:**
   - Статистика игроков сохранена
   - История боя записана
   - Таблица лидеров обновлена

4. **Телепортация возврата:**
   - Игроки возвращаются на предыдущую локацию
   - Инвентарь восстановлен
   - Эффекты сохранены

### 9. Награды

**Награды после боя:**

**Награды за достижения** (если настроены):
- 10 побед
- 25 побед
- 50 побед
- 100 побед

**Уведомление:**
Игроки получают сообщения о:
- Изменении Elo
- Новой позиции в рейтинге
- Наградах за достижения (если разблокированы)

## Форматы боя

CobbleRanked поддерживает несколько форматов боя с независимыми рейтингами для одиночных, парных, тройных, мульти и случайных форматов.

Каждый формат боя имеет:
- **Отдельные рейтинги Elo**
- **Независимые таблицы лидеров**
- **Статистику для конкретного формата** (победы, поражения, матчи)
- **Отдельные очереди**

**Победа в одиночных не влияет на ваш Elo в парных.**

### Доступные форматы

#### Конкурентные форматы

| Формат | Описание | Размер команды | Размер группы | Рейтинговый |
|--------|-------------|-----------|------------|--------|
| **SINGLES** | Бои 1v1 | 3-6 покемонов | 1 игрок | ✅ |
| **DOUBLES** | Бои 2v2 | 4-6 покемонов | 1 игрок | ✅ |
| **TRIPLES** | Бои 3v3 | 5-6 покемонов | 1 игрок | ✅ |
| **MULTI** | Командные бои 2v2 | 1-3 покемона каждому | 2 игрока | ✅ |

#### Форматы случайных боёв

| Формат | Описание | Размер команды | Поколение | Рейтинговый |
|--------|-------------|-----------|------------|--------|
| **RANDOM_SINGLES** | Случайные 1v1 | 6 покемонов (авто) | На основе пула | ✅ |
| **RANDOM_DOUBLES** | Случайные 2v2 | 6 покемонов (авто) | На основе пула | ✅ |
| **RANDOM_3V3** | Быстрые случайные | 3 покемона (авто) | На основе пула | ✅ |

### Одиночные

Традиционные бои покемонов 1v1.

**Правила:**
- Возьмите 3-6 покемонов
- Формат 1v1
- Выберите ведущего покемона перед боем

**Пример:** Игрок A против Игрока B (соло)

### Парные

1 игрок контролирует 2 покемонов одновременно.

**Правила:**
- Возьмите 4-6 покемонов
- Оба покемона на поле одновременно
- Выберите 2 ведущих перед боем

**Пример:** Игрок A (2 покемона) против Игрока B (2 покемона)

### Мульти (2v2 одиночные)

Командные бои - 2 игрока против 2 игроков.

**Правила:**
- Каждый игрок берёт 1-3 покемона
- Партнёры должны быть в одной группе
- Оба игрока выбирают ведущих

**Пример:** Группа (Игрок A + Игрок B) против Группы (Игрок C + Игрок D)

#### Как встать в очередь для Мульти

1. Сформируйте группу с партнёром
2. Оба игрока открывают GUI `/ranked`
3. Выберите формат "Multi"
4. Оба должны нажать на очередь

**Матчмейкинг:** Система создаёт пары из двух групп со схожим суммарным Elo.

### Выбор формата

Игроки выбирают формат перед постановкой в очередь:

1. Откройте GUI `/ranked`
2. Нажмите на выбор формата (Одиночные/Парные/Мульти)
3. Нажмите "Присоединиться к очереди"

### Независимая статистика

Каждый формат отслеживается отдельно:

| Статистика | По формату |
|------|-----------|
| Рейтинг Elo | ✅ |
| Победы | ✅ |
| Поражения | ✅ |
| Серия побед | ✅ |
| Всего матчей | ✅ |

**Пример:**
- Одиночные: 1500 Elo, 50 побед, 30 поражений
- Парные: 1000 Elo, 5 побед, 5 поражений
- Мульти: 1200 Elo, 20 побед, 15 поражений

### Таблицы лидеров форматов

Каждый формат имеет отдельную таблицу лидеров.

**Доступ:**
1. GUI `/ranked`
2. Нажмите "Таблицы лидеров"
3. Выберите формат

**Отображение:**
- Лучшие игроки по Elo
- Рейтинги для конкретного формата
- Фильтр по Одиночным/Парным/Мульти

### Награды для конкретных форматов

Сезонные награды и награды за достижения могут быть специфичными для формата.

**Пример:**
```json5
{
  "season_rewards": {
    "singles": { /* Награды одиночных */ },
    "doubles": { /* Награды парных */ }
  }
}
```

Игроки могут получать награды во **всех форматах**.

См. [Система наград](../configuration/rewards.md) для конфигурации.

## Обработка отключений

### Намеренное отключение (Rage Quit)

**Обнаружение:**
- Игрок закрывает игру во время боя
- Клиент отключается от сервера
- Кнопка сдачи не используется

**Штраф:**
- Счётчик побегов +1
- Считается как поражение
- Применяется штраф Elo
- Противник автоматически побеждает

**Отслеживание счётчика побегов:**
- Виден в статистике игрока
- Никогда не уменьшается автоматически
- Админ может сбросить: `/rankedadmin flee reset <игрок>`

### Ненамеренное отключение (Краш/Интернет)

**Тот же штраф:**
- CobbleRanked не может различить намеренное и ненамеренное
- Все отключения обрабатываются одинаково
- Предотвращает злоупотребления

**Защита:**
- Админ может вручную сбросить счётчик побегов для легитимных крашей
- Elo можно вручную скорректировать: `/rankedadmin elo add <игрок> <количество>`

### Проблемы с подключением

**До начала боя:**
- Игрок удаляется из очереди
- Без штрафа
- Может снова присоединиться к очереди

**Во время боя:**
- Обрабатывается как отключение
- Счётчик побегов увеличивается
- Противник побеждает

## Условия ничьей

### Достигнут лимит ходов

**Триггер:** Бой достигает максимального количества ходов (по умолчанию: 100)

**Результат:**
- Бой заканчивается ничьей
- Без изменения Elo для обоих игроков
- Не считается как победа или поражение
- Без штрафа счётчика побегов

**Конфигурация:**
```json5
"battle": {
  "maxTurns": 100  // Увеличьте для более длинных боёв
}
```

## Статистика боя

### Отслеживается по формату

- **Всего боёв:** Победы + поражения
- **Процент побед:** Победы / (победы + поражения) * 100%
- **Текущий Elo:** Рейтинг в реальном времени
- **Пиковый Elo:** Наивысший достигнутый Elo (будущая функция)
- **Счётчик побегов:** Штрафы за отключение

### Глобальная статистика

- **Всего боёв:** Сумма по всем форматам
- **Любимый формат:** Самый играемый
- **Лучший формат:** Наивысший Elo
- **Общий процент побед:** Комбинированный процент побед

## Конкурентные возможности

### Масштабирование уровня

**Принудительный лимит уровня:**
```json5
"battle": {
  "levelMatch": 70,        // Все покемоны становятся 70 уровня
  "forceLevelCap": true    // Принудительное масштабирование
}
```

**Преимущества:**
- Честная конкуренция
- Нет преимущества от фарма
- Бои, основанные на навыках

## Команды боя

### Во время боя

Специальные команды не требуются - используйте стандартный UI боя Cobblemon:
- Нажмите на приёмы для атаки
- Нажмите на покемона для замены
- Используйте предметы (если разрешено)
- Кнопка сдачи для капитуляции

### До боя

Команды перед началом боя:

```bash
/ranked              # Открыть GUI
/queue join singles  # Присоединиться к очереди
/queue leave         # Покинуть очередь
```

### После боя

Команды после боя:

```bash
/stats               # Посмотреть свою статистику
/leaderboard        # Проверить таблицу лидеров
/elo                # Проверить свой Elo
```

## Устранение неполадок

### Бой не начинается

**Симптом:** Матч найден, но бой не начинается

**Причины:**
- Невалидная команда покемонов
- Нарушение черного списка
- Ошибка конфигурации арены

**Решение:**
1. Проверьте сообщение валидации черного списка
2. Убедитесь, что арена существует: `/rankedadmin arena list`
3. Проверьте консоль на наличие ошибок

### Elo не обновляется

**Симптом:** Бой закончился, но Elo не изменился

**Причины:**
- Ничья (лимит ходов)
- Ошибка подключения к базе данных
- Ошибка конфигурации системы Elo

**Решение:**
1. Проверьте, закончился ли бой ничьей
2. Проверьте подключение к базе данных
3. Проверьте консоль на наличие ошибок
