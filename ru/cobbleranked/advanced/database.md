# Конфигурация базы данных

Продвинутая конфигурация базы данных для CobbleRanked.

## Обзор

CobbleRanked поддерживает три системы баз данных:
- **SQLite** - На основе файлов, один сервер (по умолчанию)
- **MySQL** - Сетевая база данных, поддержка cross-server
- **MongoDB** - NoSQL база данных, поддержка cross-server (НОВОЕ!)

## SQLite (по умолчанию)

### Когда использовать

**Идеально для:**
- Одиночного сервера Minecraft
- Малой и средней базы игроков (< 1,000 игроков)
- Простая настройка, без внешних зависимостей
- Тестирование и разработка

**Ограничения:**
- Не может делиться данными между несколькими серверами
- Производительность деградирует с очень большими наборами данных (10,000+ игроков)

### Конфигурация

Конфигурация не требуется! Работает из коробки.

**Расположение файла:** `config/cobbleranked/ranked.db`

### Резервное копирование

```bash
# Резервное копирование
cp config/cobbleranked/ranked.db backup-$(date +%Y%m%d).db

# Восстановление
cp backup-20251026.db config/cobbleranked/ranked.db
```

## MySQL

### Когда использовать

**Требуется для:**
- Настроек cross-server (несколько серверов Minecraft)
- Больших баз игроков (1,000+ игроков)
- Лучшей производительности в масштабе
- Продвинутых функций (репликация, инструменты резервного копирования)

### Настройка

См. [Настройка Cross-Server](cross-server.md) для полного руководства по установке MySQL.

**Быстрая конфигурация:**
```json5
{
  "cross_server": {
    "enabled": true,
    "mysql": {
      "host": "localhost",
      "port": 3306,
      "database": "cobbleranked",
      "username": "cobbleranked",
      "password": "secure_password"
    }
  }
}
```

## MongoDB

### Когда использовать

**Идеально для:**
- Настроек cross-server (альтернатива MySQL)
- Облачных развёртываний (MongoDB Atlas)
- Требований горизонтального масштабирования
- Гибкой эволюции схемы
- Больших баз игроков с высокой пропускной способностью записи

### Настройка

#### Вариант 1: Локальный MongoDB

```bash
sudo apt install mongodb-server
sudo systemctl start mongodb
```

#### Вариант 2: MongoDB Atlas (Облако)

1. Создайте бесплатный кластер на [mongodb.com/cloud/atlas](https://www.mongodb.com/cloud/atlas)
2. Получите строку подключения
3. Добавьте IP-адрес в белый список
4. Скопируйте учётные данные

### Конфигурация

```json5
{
  "cross_server": {
    "enabled": true,
    "database_type": "mongodb",
    "mongodb": {
      "connection_string": "mongodb://localhost:27017",
      "database": "cobbleranked",
      "username": "cobbleranked",
      "password": "secure_password",
      "use_srv": false
    }
  }
}
```

## Сравнение

| Функция | SQLite | MySQL | MongoDB |
|---------|--------|-------|---------|
| **Настройка** | ✅ Нулевая конфигурация | ⚠️ Требуется установка | ⚠️ Требуется установка |
| **Cross-server** | ❌ Нет | ✅ Да | ✅ Да |
| **Масштабирование** | ❌ Ограниченное | ⚠️ Только вертикальное | ✅ Горизонтальное |
| **Облако** | ❌ Нет | ⚠️ Требуется VPS | ✅ Бесплатный уровень Atlas |

**Рекомендация:**
- **Один сервер**: SQLite (самый простой)
- **2-5 серверов**: MySQL или MongoDB (похожие)
- **5+ серверов**: MongoDB (лучшее масштабирование)
- **Облачное развёртывание**: MongoDB Atlas (самый простой)
