# Глоссарий

Технические термины и концепции, используемые в документации CobbleRanked.

---

## Системы рейтинга

### Рейтинг Эло
Числовая система ранжирования навыков, которая рассчитывает силу игрока на основе результатов матчей. В CobbleRanked игроки начинают с 1000 Эло по умолчанию.

**Связанные страницы:** [Система Эло](features/elo-system.md)

### K-фактор
Коэффициент, который определяет, насколько сильно изменяется Эло за матч. Больший K-фактор = более быстрые изменения рейтинга.
- Новые игроки (< 10 матчей): K = 64
- Опытные игроки: K = 32

**Связанные страницы:** [Конфигурация системы Эло](configuration/config.md#elo-system)

### RD (Отклонение рейтинга)
В системе Glicko-2 представляет неопределенность в рейтинге игрока. Более высокий RD = менее определенный рейтинг.
- Новые игроки: RD = 350
- Опытные игроки: RD ≈ 50-100
- Увеличивается при неактивности

**Связанные страницы:** [Режим Glicko-2](features/elo-system.md#glicko-2-advanced)

### Испытательные матчи
Первые 10 матчей для новых игроков, где Эло изменяется более драматично (K-фактор удвоен).

**Связанные страницы:** [Руководство по K-фактору](features/elo-system.md#k-factor-guide-pokemon-showdown-mode)

### Распад Эло
Система, которая со временем снижает Эло неактивных игроков, чтобы предотвратить застой в рейтинговой таблице.
- По умолчанию: -2 Эло в день после периода неактивности
- Выполняется ежедневно в 9:00 UTC

**Связанные страницы:** [Конфигурация распада Эло](features/elo-system.md#elo-decay)

## Система сражений

### Правила сражений
Правила, применяемые для обеспечения честных соревновательных сражений:

#### Species Clause
Предотвращает дублирование покемонов в одной команде (по умолчанию: включено).

#### OHKO Clause
Запрещает атаки с одним ударом нокаута, такие как Fissure, Sheer Cold (по умолчанию: включено).

#### Sleep Clause
Предотвращает одновременный сон 2+ покемонов противника (по умолчанию: включено).

#### Evasion Clause
Запрещает атаки, повышающие уклонение (Minimize, Double Team) (по умолчанию: включено).

#### Item Clause
Предотвращает дублирование удерживаемых предметов в одной команде (по умолчанию: отключено).

#### Dynamax Clause
Запрещает Dynamax/Gigantamax (по умолчанию: включено).

**Связанные страницы:** [Форматы сражений](features/ranked-battles.md#battle-clauses)

### Атаки OHKO
Атаки с одним ударом нокаута, которые обходят обычный расчет урона:
- Fissure
- Sheer Cold
- Horn Drill
- Guillotine

**Связанные страницы:** [Конфигурация черного списка](configuration/blacklist.md)

### Форматы сражений

#### Одиночные (1v1)
- Команда: 3-6 покемонов
- Активные: 1 против 1
- Независимый рейтинг Эло

#### Парные (2v2)
- Команда: 4-6 покемонов
- Активные: 2 против 2
- Формат в стиле VGC
- Независимый рейтинг Эло

#### Командное сражение (2v2 команды)
- Команда: 1-3 покемона на игрока
- Активные: 2 против 2
- Общее Эло с членами группы

**Связанные страницы:** [Форматы сражений](features/ranked-battles.md)

## Серверные системы

### Арена
Назначенное место для сражений с сохраненными координатами и направлением взгляда. Игроки телепортируются на арены при начале матчей.

**Команда:** `/rankedadmin arena set <name>`

**Связанные страницы:** [Настройка арен](configuration/arenas.md)

### Штраф за побег
Временный бан от подбора противников за отказ от сражений:
- 1-5 побегов: бан на 5 минут
- 6-10 побегов: бан на 15 минут
- 11+ побегов: бан на 30 минут
- Распад: -1 побег каждые 24 часа

**Связанные страницы:** [Штрафы за отключение](features/disconnect-penalties.md)

### Кросс-сервер
Многосерверная архитектура, разделяющая рейтинги по всей сети:
- Требуется: MySQL/MongoDB + Redis
- Разделяет серверы лобби от серверов сражений
- Синхронизирует Эло в реальном времени

**Связанные страницы:** [Настройка кросс-сервера](advanced/cross-server.md)

### Диапазон подбора противников
Разница в Эло, допустимая при поиске противников:
- Начальная: ±200 Эло
- Расширяется через 30 секунд
- Максимум: ±600 Эло (множитель 3.0)

**Связанные страницы:** [Система Эло](features/elo-system.md)

## Конфигурация

### JSON5
Расширенный формат JSON, допускающий комментарии (`//` и `/* */`). Все конфигурационные файлы CobbleRanked используют JSON5.

**Пример:**
```json5
{
  // Это комментарий
  "initialElo": 1000,  // Разрешена завершающая запятая
}
```

**Связанные страницы:** Все страницы конфигурации

### Черный список
Система для ограничения покемонов, атак, способностей и предметов в рейтинговых сражениях.

**Категории:**
- Метки покемонов (legendary, mythical и т.д.)
- Конкретные покемоны (Mewtwo, Rayquaza)
- Атаки (Baton Pass, Last Respects)
- Способности (Moody, Shadow Tag)
- Предметы (King's Rock, Berserk Gene)

**Связанные страницы:** [Конфигурация черного списка](configuration/blacklist.md)

### Метка
Теги категорий покемонов, используемые для черного списка:
- `legendary` - Легендарные покемоны
- `mythical` - Мифические покемоны
- `restricted` - Ограниченные покемоны VGC
- `ultra_beast` - Ультра звери
- `paradox` - Парадокс покемоны
- `sub_legendary` - Младшие легендарные

**Связанные страницы:** [Категории меток черного списка](configuration/blacklist.md#label-categories)

### Ограничения по меткам
Ограничения в стиле VGC, допускающие ограниченное использование вместо полного запрета.

**Пример:** Разрешить максимум 2 ограниченных легендарных покемона в команде
```json5
{
  "restricted_label_limits": {
    "restricted": 2
  }
}
```

**Связанные страницы:** [Ограничения по меткам](configuration/blacklist.md#label-limits-vgc-style)

## Система сезонов

### Сезон
Ограниченный по времени соревновательный период (по умолчанию: 30 дней), после которого рейтинги сбрасываются.

**Функции:**
- Автоматическая ротация
- Награды для топ-3 игроков
- Сброс Эло до начального значения
- Сброс статистики побед/поражений

**Связанные страницы:** [Сезоны](features/seasons.md)

### Сезонные награды
Призы, выдаваемые игрокам с высоким рейтингом в конце сезона, настраиваются для каждого формата (Одиночные/Парные/Командные).

**Пример:**
```json5
{
  "rewards": {
    "1": ["give %player% diamond 64"],
    "2": ["give %player% emerald 48"],
    "3": ["give %player% gold_ingot 32"]
  }
}
```

**Связанные страницы:** [Конфигурация наград](configuration/rewards.md)

### Награды за достижения
Награды за достижения, основанные на количестве побед или рейтинге Эло.

**Пример:**
```json5
{
  "win_milestones": {
    "10": ["give %player% rare_candy 5"],
    "50": ["give %player% master_ball 1"]
  }
}
```

**Связанные страницы:** [Награды за достижения](configuration/rewards.md#milestone-rewards)

## Технические термины

### Redis
Хранилище данных в памяти, используемое для кросс-серверной связи в реальном времени и синхронизации очередей.

**Требуется для:** Кросс-серверных настроек

**Связанные страницы:** [Конфигурация базы данных](advanced/database.md#redis-configuration)

### SQLite
Встроенная база данных по умолчанию. Хранит статистику игроков, Эло и историю матчей локально.

**Ограничения:** Только для одного сервера (нет кросс-сервера)

**Связанные страницы:** [Конфигурация базы данных](advanced/database.md)

### MySQL
Реляционная база данных для кросс-серверных настроек (рекомендуется 2-5 серверов).

**Связанные страницы:** [Конфигурация базы данных](advanced/database.md#mysql)

### MongoDB
NoSQL база данных для кросс-серверных настроек (5+ серверов, облачная).

**Связанные страницы:** [Конфигурация базы данных](advanced/database.md#mongodb)

### Velocity
Современный прокси-сервер Minecraft для подключения нескольких серверов в сети.

**Требуется для:** Кросс-серверных рейтинговых сражений

**Связанные страницы:** [Настройка кросс-сервера](advanced/cross-server.md)

## Системы игроков

### Очередь
Система подбора противников, которая объединяет игроков на основе Эло и предпочтения формата.

**Поведение:**
- Поиск в пределах диапазона Эло
- Расширение диапазона каждые 30 секунд
- Автоотмена после достижения максимального диапазона

**Связанные страницы:** [Рейтинговые сражения](features/ranked-battles.md)

### Таблица лидеров
Топ-10 рейтингов по формату в реальном времени, обновляемые после каждого матча.

**Отображение:** GUI `/ranked` или игровые команды

**Связанные страницы:** [Таблицы лидеров](features/leaderboards.md)

### Таймер хода
Необязательное ограничение времени на ход для предотвращения затягивания.

**По умолчанию:** Отключено (нет ограничения по времени)

**Связанные страницы:** [Таймер хода](features/turn-timer.md)
